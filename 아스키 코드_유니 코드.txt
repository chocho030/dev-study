아스키 코드

ASCII(American Standard Code for Information In terchange, 미국 정보 교환 표준 부호) - 미국 ANSI(미국 국가표준 협회)에서 표준화한 정보 교환용 7비트 부호 체계

대부분의 문자 인코딩의 기초

8비트 - 1비트는 통신 에러 검출을 위해 사용됨. 1비트 = parity bit. 통신 정보가 도중에 변질된 것을 수신 측에서 알아낼 확률을 높이는 데 사용

7비트 인코딩과 33개의 출력 불가능한 제어 문자들과 공백을 포함한 95개의 출력 가능한 문자들로 구성되어 총 128개로 이뤄짐
-
영문 알파벳과 키보드에 있는 문자들을 정리한 대표적인 문자 인코딩 방식
총 128개

대문자 - 65 - 90
소문자 - 97 - 122

문자를 8비트, 이진수로 표현

시스템 간의 데이터 교환과 통신에 유용

다른 언어의 문자 발음 기호 특수 문자 등은 표현 불가
-
ASCII 코드는 문자를 컴퓨터가 이해할 수 있는 이진 데이터(0 or 1)로 변환하는 표준 방법.
-
미국 국립 표준 협회 (ANSI)애서 표준화한 정보교환용 7비트 부호체계

ASCII란 말 자체가 미국정보교환표준부호의 줄임말.

ASCII라는 말에 이미 코드가 있기 때문에 아스키 코드라는 말에는 어폐가 있음.

영문 알파벳을 이용하는 대표적인 문자 인코딩. (문자 인코딩 - 인코딩은 사용자가 입력한 문자나 기호들을 컴퓨터가 이용할 수 있는 신호로 만드는 것을 말함)

컴퓨터와 통신 장비를 비롯한 문자를 사용하는 많은 장치에서 사용되며, 대부분의 문자 인코딩이 아스키애 기초를 두고 있음.

33개릐 출력 불가한 제어 문자와 공잭을 미롯한 95개의 출력 가능한 문자로 총 128개로 이루어져 있음.

제어 문자들은 역사적 이유롷 남아 있으나 대부분 더 이상 사용되지 않는다

패리티 검사
패리티 코드 : 8비트 중 통신 에러 검출을 위해 사용하는 1비트를 이르는 것  > 패리티 고드를 확인함으로써 전송 도중 변질된 것을 수신측에서 검출해내는 것을 패리티 검사라고 함.
패리티 검사에는 짝수 패리티와 홀수 패리티가 있으며 오버플로우를 이용한 체크섬 방식이다.

-

아스키 기반의 확장 인코딩들이 등장. 이들을 묶어서 아스키라고 부르기도 함.

대표적으로 7비트 인코딩을 유지한 ISO/IEC 646과, 원래 아스키 앞에 비트 0을 넣어 8비트 인코딩을 만든 IBM 코드 페이지, ISO 8859 있음

유니 코드

전세계 모든 문자를 처리하기 위한 국제적 표준 문자 전산 처리법

언어 표기 문자 뿐만 아니라 악보 기호, 이모지, 탸크 등도 표현 가능

8비트 > 16비트로 확대

코드가 더 빨리 수행, 더 적은 메모리 > 효율성 증대, 리소스 처리 용이, com과 상호운용이 쉽다는 장점

-

전 세계의 모든 문자를 컴퓨터에서 일관되게 표현하고 다룰 수 있도록 설계된 산업 표준

유니코드는 ASCII를 확장한 형태

 ISO 10646 문자 집합, 문자 인코딩, 문자 정보 데이터베이스, 문자를 다루기 위한 알고리즘 등을 포함

-

다양한 인코딩 방식이 있는데, UTF-8, DTF-16,EEUC-KR 등등 있음.

UTF-8 : 유니코드를 인코딩하는 방식으로 8비트 단위로 가변 인코딩함.
유니코드의 시작 바이트를 보고 몇 바이트를 읽을 것인지를 결정하기로 약속.

EUC-KR : 한글을 표현하기 위해 (16비트)로 사용
